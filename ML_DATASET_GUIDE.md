# Документація: Збір даних для Machine Learning (ML Dataset)

Цей документ описує архітектуру системи "Human-in-the-loop", яку ми використовуємо для покращення алгоритму автоматичного розпізнавання карток.

## 🎯 Мета
Поточний алгоритм розпізнавання (заснований на класичних методах комп'ютерного зору OpenCV) працює непогано, але не ідеально. Часто користувачу доводиться вручну підправляти кути розпізнаних карток. 
Замість того, щоб ці виправлення пропадали марно, ми зберігаємо їх. Коли назбирається достатньо даних (наприклад, 200-500 сканів), ми зможемо натренувати власну нейромережу (наприклад, YOLOv8 або сегментаційну модель), яка розпізнаватиме картки набагато точніше.

---

## 🏗 Як відбувається збір даних

Коли користувач натискає кнопку **Download** (за умови ввімкненого чекбоксу "Share data for AI"), відбувається наступне:

1. **Frontend (`js/cards/telemetry.js`)**:
   - Бере поточний скан із Canvas.
   - Стискає його до прийнятних для ML розмірів (максимум 1000 пікселів по більшій стороні), щоб зекономити трафік та місце.
   - **Нормалізує координати**: замість абсолютних пікселів (напр., `X: 300px`), координати кутів перераховуються у відсотки від `0.0` до `1.0` відносно ширини та висоти картинки. Це робить координати незалежними від розміру зображення.
   - Відправляє POST запит у фоновому режимі (не блокуючи користувача) на наш бекенд.

2. **Backend (`netlify/functions/save-dataset.js`)**:
   - Отримує зображення (base64) та завантажує його у хмарне сховище **Cloudinary** (в папку `cut_image_telemetry`).
   - Отримує від Cloudinary пряме посилання (`secure_url`) на збережене зображення.
   - Записує у **MongoDB Atlas** (база `cut_image_ai`, колекція `telemetry`) об'єкт із даними:
     - `timestamp`: час збереження.
     - `mode`: режим (наприклад, 'detect' або 'rect').
     - `dimensions`: оригінальні розміри зображення.
     - `coordinates`: масив нормалізованих координат (4 кути для кожної картки).
     - `imageUrl`: пряме посилання на картинку в Cloudinary.

## 🚀 Як використовувати ці дані в майбутньому

Коли в базі MongoDB накопичиться достатньо записів (хоча б 100-200 сканів із виправленими координатами), ви зможете використати їх для тренування моделі.

### Крок 1: Експорт датасету (Запуск скрипта)
Ми вже підготували готовий скрипт для цього! Перейдіть у терміналі в папку `scripts` і виконайте наступні команди (переконайтеся, що у вас встановлений Python):

```bash
cd scripts
pip install -r requirements.txt
python download_dataset.py
```

Цей скрипт автоматично:
1. Підключиться до бази `telemetry`.
2. Завантажить усі нові картинки за посиланням `imageUrl` у папку `dataset/images/`.
3. Згенерує текстові файли розмітки у форматі YOLO-OBB у папку `dataset/labels/`.

### Крок 2: Конвертація розмітки (Annotations)
Нейромережі (наприклад, архітектури YOLO) вимагають розмітку в специфічному текстовому форматі.
Оскільки ми зберегли нормалізовані координати `[0.0 - 1.0]`, їх дуже легко конвертувати:

*Якщо ви використовуєте класичну Bounding Box модель (наприклад, старі версії YOLO):*
Вона поверне лише прямий прямокутник (без нахилу). Це **НЕ ПІДХОДИТЬ** для нашої задачі, оскільки скани часто повернуті або мають перспективне спотворення (skew).

*Правильні підходи для нашої задачі (робота з 4 кутами):*
1. **Keypoint Detection (YOLOv8-Pose)**: Замість того, щоб шукати "прямокутник", модель навчається знаходити 4 конкретні точки (кути картки). Це ідеальний підхід, бо він напряму видає координати кутів, які потрібні для вирівнювання перспективи (perspective transform).
2. **Oriented Bounding Boxes (YOLOv8-OBB)**: Спеціальний різновид моделей, який знаходить повернуті прямокутники (додається кут нахилу).
3. **Instance Segmentation (YOLOv8-Seg)**: Модель виділяє кожен піксель картки, після чого за допомогою OpenCV (функція `minAreaRect` або `approxPolyDP`) ми знаходимо 4 кути знайденого багатокутника.

Оскільки ми зберегли точні 4 кути (polygons), ці координати ідеально підходять для конвертації у формат **Keypoint Detection** або **Segmentation**.

### Крок 3: Навчання (Training)
1. Розділити збережені дані на Training Set (80%) та Validation Set (20%).
2. Використати фреймворк (наприклад, Ultralytics YOLO або PyTorch).
3. Натренувати модель виявляти клас `card` на сканах.

### Крок 4: Інтеграція на Frontend
Після успішного навчання модель можна експортувати у формат **ONNX** або **TensorFlow.js**. 
Це дозволить завантажувати нейромережу прямо в браузер користувача (як ми зараз робимо з OpenCV) і виконувати передбачення локально, без необхідності тримати дорогі GPU-сервери для бекенду. 

Таким чином, "ручна" робота користувачів зараз дозволить повністю автоматизувати цей процес у майбутньому!
