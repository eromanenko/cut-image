# Інструкція з навчання ML моделі (YOLOv8-OBB)

Ця інструкція описує практичні кроки для навчання власної моделі розпізнавання карток на основі даних, зібраних системою "Human-in-the-loop" (як описано у `ML_DATASET_GUIDE.md`).

## 1. Завантаження та підготовка датасету

Оскільки наш застосунок вже збирає дані, ручна розмітка не потрібна. У нас вже є скрипт для завантаження зображень та конвертації координат у формат **YOLO-OBB** (Oriented Bounding Boxes - орієнтовані/нахилені прямокутники).

Відкрийте термінал у корені проєкту та виконайте команди:

```bash
cd scripts
pip install -r requirements.txt
python download_dataset.py
```

**Результат:** На вашому диску з'явиться папка `dataset/`, що містить підпапки `images/` та `labels/` з готовими для навчання даними. Також поруч має бути створений конфігураційний файл `dataset.yaml`.

## 2. Вибір обладнання (Hardware)

Для навчання моделі комп'ютерного зору потрібна відеокарта (GPU):

*   **Локально:** Якщо у вас є відеокарта NVIDIA (від RTX 2060 / 8GB VRAM і вище), ви можете навчати модель прямо на своєму комп'ютері. Це швидко і зручно.
*   **Google Colab:** Якщо потужної відеокарти немає, заархівуйте папку `dataset/` (та файл `dataset.yaml`) у ZIP-архів. Завантажте цей архів у безкоштовний сервіс [Google Colab](https://colab.research.google.com/), розпакуйте там і виконуйте кроки навчання на хмарних GPU.

## 3. Навчання першої моделі (Baseline)

Ми використовуємо архітектуру **YOLOv8-OBB**, оскільки вона вміє знаходити об'єкти під кутом, що критично важливо для сканів карток з викривленою перспективою.

1. Встановіть необхідну бібліотеку Ultralytics:
   ```bash
   pip install ultralytics
   ```

2. Запустіть процес навчання (налаштуйте шлях до файлу dataset.yaml):
   ```bash
   yolo task=obb mode=train data=шлях/до/dataset.yaml model=yolov8n-obb.pt epochs=50 imgsz=1024
   ```
   
   **Пояснення параметрів:**
   - `model=yolov8n-obb.pt`: Використовуємо найменшу та найшвидшу модель (версія "nano") як базову точку для старту (Transfer Learning).
   - `epochs=50`: Кількість проходів по всьому датасету. Якщо модель перестане вчитися швидше (перестане падати помилка), процес автоматично зупиниться (Early Stopping).
   - `imgsz=1024`: Розмір зображень для тренування (сканів). Важливо, щоб він приблизно співпадав з розміром картинок, які ми збираємо.

**Результат:** Після завершення навчання, ваги вашої найкращої моделі будуть збережені у файл `runs/obb/train/weights/best.pt`. Це і є "мозок" нашої системи розпізнавання.

## 4. Підготовка моделі для Frontend (Експорт)

Файл `best.pt` призначений для роботи в Python. Щоб наша модель працювала прямо в браузері користувача (як ми зараз робимо з OpenCV.js), її потрібно експортувати:

```bash
# Експорт для ONNX Runtime Web
yolo export model=runs/obb/train/weights/best.pt format=onnx

# Або експорт для TensorFlow.js
yolo export model=runs/obb/train/weights/best.pt format=tfjs
```

Отриманий файл (наприклад, `best.onnx`) ми зможемо підключити до нашого `index.html` за допомогою JavaScript, щоб повністю замінити поточний застарілий алгоритм детекції.

## 5. Безперервне навчання (Continuous / Active Learning)

Після впровадження нової AI-моделі у веб-застосунок, збір телеметрії ("Share data for AI") змінює свій формат на **Active Learning (Активне навчання)**.

### 5.1. Збір лише складних прикладів (Hard Example Mining)
Збір телеметрії не вимикається, але оптимізується. Телеметрія (координати та зображення) відправляється до MongoDB **ТІЛЬКИ у тому випадку, якщо користувач вручну відредагував результати автодетекту** (наприклад, пересунув кут або додав пропущену картку). 
Якщо модель відпрацювала ідеально і користувач нічого не чіпав — дані не відправляються. Це гарантує, що база даних не буде "роздуватися" від простих прикладів, а наповнюватиметься виключно найціннішими, складними випадками (edge cases).

### 5.2. Протидія катастрофічному забуванню
Для донавчання **необхідно змішувати старі дані з новими**. Якщо тренувати модель тільки на нових складних прикладах, вона швидко навчиться розпізнавати їх, але може "забути", як виглядають звичайні прості картки (Catastrophic Forgetting). Скрипт завантаження даних спеціально спроєктований так, щоб витягувати всю накопичену базу — це правильний підхід.

### 5.3. Як донавчати модель у майбутньому:
Коли накопичиться ще ~200 нових виправлень у MongoDB, ви повторюєте завантаження даних (Крок 1). Далі, під час Кроку 3 замість базової `yolov8n-obb.pt` ви вказуєте **вашу поточну найкращу навчену модель**:

```bash
yolo task=obb mode=train data=dataset.yaml model=шлях_до_ВАШОГО_старого_best.pt epochs=30
```
Завдяки цьому модель закріпить старі знання та вивчить нові складні випадки, стаючи ще точнішою!
